# Domain Prompt — Monopoly Phase 1 UI Flow

## 领域边界

### HUD 常驻信息
- 当前玩家名称/颜色标识
- 所有玩家的资金余额
- 当前回合数
- 每人拥有的地产数量
- 当前游戏阶段状态

### 决策型弹窗（需要玩家操作）
1. **掷骰弹窗** — 回合开始时，[掷骰子] 按钮
2. **购买弹窗** — 踩到无主地产时，[购买 $X] / [放弃] 按钮
3. **监狱决策弹窗** — 在狱中回合开始时，[支付 $50 保释] / [尝试掷双数] 按钮

### 信息型弹窗（仅展示确认）
4. **租金弹窗** — 踩到他人地产时，"支付租金 $X 给玩家Y" [确认]
5. **税务弹窗** — 踩到税务格时，"支付税金 $X" [确认]
6. **破产弹窗** — 资金不足时，"玩家X 破产退出！" [确认]
7. **胜利弹窗** — 仅剩一人时，"玩家X 获胜！" [再来一局]

### UI 与状态绑定
- HUD 实时反映 GameState/PlayerState 数据
- 弹窗由 GameMode 的状态转换触发
- 按钮回调通知 GameMode 继续流程
- MPlayerController 统一管理 Widget 生命周期
