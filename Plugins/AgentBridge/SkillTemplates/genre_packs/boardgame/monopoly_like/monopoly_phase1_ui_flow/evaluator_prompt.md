# Evaluator Prompt — Phase 1 UI Flow 自检

## 必过检查项

1. **决策节点覆盖**：是否为所有需要玩家操作的游戏节点都定义了弹窗？
   - 掷骰（每回合）
   - 购买地产（踩到无主地产）
   - 监狱决策（在狱中回合开始）
2. **信息提示覆盖**：是否为所有需要告知玩家的事件都定义了弹窗？
   - 支付租金
   - 支付税金
   - 破产通知
   - 胜利通知
3. **HUD 完整性**：HUD 是否显示当前玩家、所有玩家资金、回合数？
4. **按钮动作明确**：每个按钮的 action 是否对应一个明确的游戏流程回调？
5. **触发条件精确**：每个弹窗的 trigger 是否与 turn_flow / property_economy / jail 的状态一致？
6. **状态绑定完整**：HUD 数据是否绑定到正确的 GameState/PlayerState 字段？
7. **Phase 范围**：是否有面向 Phase 2 在线多人的 UI 元素？
