# Domain Prompt — Monopoly Board Topology

## 领域边界

本 Skill 只处理棋盘的 **拓扑结构**，即：
- 棋盘形状（正方形环形）
- 总格数（28 格）
- 每边格数（7 格，含两端角格）
- 移动方向（顺时针）
- 四个角格的固定索引（起点 0、监狱探视 7、免费停车 14、前往监狱 21）

## 范围约束

### 必须处理
- 确认棋盘为闭合环形路线
- 确认格子索引 0..27 无缺口
- 确认四角格位置固定且索引正确
- 确认移动方向为单向顺时针

### 不处理
- 格子的类型、价格、租金等属性（由 tile_event_dispatch 处理）
- 回合流转、骰子规则（由 turn_and_dice_flow 处理）
- Phase 2 在线多人的网络拓扑
- 棋盘的 3D 世界坐标和视觉布局（由 Lowering 阶段处理）

## 常见错误预防
- 不要把 28 格误解为 40 格（标准大富翁是 40 格，本 GDD 是简化版 28 格）
- 不要假设可以动态修改棋盘结构
- 角格索引必须精确匹配 GDD 定义，不得自行调整
